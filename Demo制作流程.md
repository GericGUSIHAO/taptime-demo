# Demo 制作流程详解

> 目标：用 Vibe Coding 做一个顶级设计师级别的网页 Demo
> 风格：Editorial Magazine 极简风 + 仪式感 + 微动效
> 参考：Nosh（日记本+贴纸）、HabitDone（简洁克制）、设计师品牌官网

---

## 一、整体流程概览

```
设计规划 → 静态原型 → 动效设计 → 功能实现 → 优化打磨 → 部署上线
```

| 阶段 | 任务 | 输出 | 预计时间 |
|---|---|---|---|
| **设计规划** | 确定界面结构、视觉风格、交互方式 | 设计文档 | 0.5天 |
| **静态原型** | 用 TRAE 生成静态界面 | HTML/CSS | 1天 |
| **动效设计** | 添加过渡、悬停、点击反馈动画 | 动效代码 | 0.5天 |
| **功能实现** | 时间块记录、习惯打卡逻辑 | JS 逻辑 | 1-2天 |
| **优化打磨** | 细节调整、响应式适配 | 最终 Demo | 0.5天 |
| **部署上线** | GitHub Pages / Vercel | 体验链接 | 0.5天 |

## 二、阶段一：设计规划

> 这个阶段要先想清楚 Demo 有哪些界面、长什么样、用户怎么操作。
> 不是直接写代码，而是先画"设计蓝图"。

### 2.1 界面结构规划

TapTime Demo 需要的界面：

| 界面 | 内容 | 用户操作 |
|---|---|---|
| **首页（Today）** | 今日时间块卡片、今日习惯列表 | 点击卡片记录时间、点击习惯打卡 |
| **时间块页面** | 本周时间分布可视化 | 查看、切换日期 |
| **习惯页面** | 习惯列表 + 状态（建立期/稳定期） | 切换习惯状态 |
| **设置页面** | 添加/编辑习惯 | 设置习惯参数 |

**建议初赛简化为 2 个界面**：
- 首页（Today）：时间块记录 + 习惯打卡
- 时间块页面：本周时间分布

### 2.2 视觉风格确定

已确定的设计方向：

| 元素 | 设计决策 |
|---|---|
| **整体风格** | Editorial Magazine 极简风 + 仪式感 |
| **配色** | 米白底（#FAF9F7）+ 纯黑文字 + 深赭石色强调（#C45C2E） |
| **字体** | Fraunces（衬线标题）+ Space Grotesk（正文） |
| **布局** | 大量留白、不对称网格、大标题 |
| **卡片设计** | 圆角卡片、微阴影、悬停动效 |
| **时间块设计** | 参考 Nosh：时间块像"印章"，点击后留下痕迹 |
| **习惯状态** | 建立期 = 打卡按钮，稳定期 = 默认已完成（只有"跳过"按钮） |

### 2.3 交互方式规划

| 交互 | 设计 | 动效 |
|---|---|---|
| **点击时间卡片** | 点1次=15min，点2次=30min | 点击时卡片缩放→恢复，数字动画增长 |
| **打卡习惯** | 点击打卡按钮 | 打勾动画，卡片轻微抬起 |
| **切换日期** | 左右滑动或点击日期 | 页面滑动过渡 |
| **添加时间块** | 点击"+"按钮 | 模态框从底部弹出 |
| **悬停效果** | 所有可点击元素 | 轻微放大 + 阴影增强 |

### 2.4 输出：设计文档

把以上规划整理成一个设计文档，包含：
- 界面结构图（手绘或截图）
- 配色方案（颜色值）
- 字体选择（名称+用途）
- 交互流程图（用户怎么操作）

## 三、阶段二：静态原型

> 这个阶段用 TRAE 生成静态界面，只做"长什么样"，不做功能。

### 3.1 TRAE 对话策略

**推荐方式**：分界面生成，而不是一次性生成整个 Demo

| 对话 | Prompt 内容 | 输出 |
|---|---|---|
| **对话1** | "生成 TapTime 首页的静态原型，包含时间块卡片和习惯打卡区域，Editorial Magazine 极简风..." | 首页 HTML/CSS |
| **对话2** | "生成时间块页面，展示本周时间分布，用简洁的可视化方式..." | 时间块页面 HTML/CSS |
| **对话3** | "生成导航和整体布局框架..." | 导航 + 响应式布局 |

### 3.2 Prompt 模板示例

**对话1：首页静态原型**

```
我要做一个效率类 App 的网页 Demo，产品名是 TapTime。

请帮我生成首页（Today）的静态原型，包含：

1. **今日时间块区域**：
   - 顶部显示"Today"
   - 显示几个预设卡片（Work、Read、Move、Rest等）
   - 每个卡片显示已记录的时间（如"45min"）
   - 点击交互：点1次=15min，点2次=30min

2. **今日习惯区域**：
   - 显示几个习惯条目（如"健身"、"阅读"、"冥想"）
   - 每个习惯有打卡按钮或"已完成"状态
   - 建立期显示打卡按钮，稳定期显示默认已完成

设计风格：
- Editorial Magazine 极简风
- 配色：米白底（#FAF9F7）、纯黑文字、深赭石色强调（#C45C2E）
- 字体：Fraunces（标题）+ Space Grotesk（正文）
- 大量留白、圆角卡片、微阴影

先只做静态界面，不需要实际功能。请生成完整的 HTML/CSS 代码。
```

### 3.3 验证设计方向

静态原型生成后：
- 用浏览器打开看效果
- 检查配色、字体、布局是否符合预期
- 如果不满意，继续对话调整

## 四、阶段三：动效设计

> 这个阶段给静态原型添加动画效果，让界面"活起来"。

### 4.1 需要添加的动效

| 动效类型 | 应用场景 | 实现方式 |
|---|---|---|
| **页面加载动效** | 首页元素依次渐入 | CSS animation + animation-delay |
| **点击反馈** | 时间卡片点击时缩放 | CSS transform + transition |
| **数字增长动画** | 时间从0增长到实际值 | CSS 或 JS 动画 |
| **打卡动效** | 打勾动画 + 卡片抬起 | CSS keyframes |
| **悬停效果** | 卡片悬停时放大+阴影 | CSS :hover + transition |
| **页面切换** | 日期切换时滑动过渡 | CSS transform + transition |

### 4.2 TRAE 对话策略

**对话示例**：

```
请给刚才生成的首页添加动效：

1. 页面加载时，所有卡片依次渐入（从上到下，间隔100ms）
2. 点击时间卡片时，卡片缩放（scale 0.95 → 1），时间数字动画增长
3. 打卡时，打勾符号动画出现（从0放大到1）
4. 所有可点击元素悬停时轻微放大（scale 1.02）

请用 CSS animation 和 transition 实现，保持代码简洁。
```

### 4.3 动效设计原则

| 原则 | 说明 |
|---|---|
| **克制** | 动效要轻，不要太花，影响专业感 |
| **功能性** | 动效要服务于交互（让用户知道点击成功了） |
| **一致性** | 所有动效保持相同的节奏（如都用 0.3s ease） |
| **流畅** | 动效不能卡顿，影响体验 |

## 五、阶段四：功能实现

> 这个阶段让 Demo "能真正用起来"，而不是只好看。

### 5.1 需要实现的功能

| 功能 | 逻辑 | 数据存储 |
|---|---|---|
| **时间块记录** | 点击卡片 → 累加时间 → 更新显示 | localStorage |
| **时间块持久化** | 刷新页面后数据不丢失 | localStorage |
| **习惯打卡** | 点击打卡 → 切换为"已完成" → 记录日期 | localStorage |
| **习惯状态切换** | 连续打卡30天后自动切换为稳定期 | localStorage |
| **日期切换** | 查看不同日期的时间块 | localStorage |

### 5.2 TRAE 对话策略

**分功能对话，而不是一次性实现所有功能**：

| 对话 | Prompt 内容 | 输出 |
|---|---|---|
| **对话1** | "实现时间块记录功能：点击卡片累加时间，用 localStorage 存储..." | 时间记录 JS |
| **对话2** | "实现习惯打卡功能：点击打卡切换状态，存储打卡日期..." | 打卡 JS |
| **对话3** | "实现习惯状态自动切换：连续打卡30天后自动变成稳定期..." | 状态切换 JS |
| **对话4** | "实现日期切换功能：可以查看不同日期的时间块..." | 日期切换 JS |

### 5.3 Prompt 模板示例

**对话1：时间块记录功能**

```
请实现时间块记录功能：

逻辑：
1. 点击卡片第1次 → 累加 15min
2. 点击卡片第2次 → 累加 30min
3. 点击卡片第3次 → 累加 45min
4. 点击卡片后数字动画增长到新值

数据存储：
- 用 localStorage 存储，键名 "taptime_blocks"
- 存储格式：{ date: "2026-06-27", blocks: { work: 45, read: 30, move: 15 } }

显示：
- 页面加载时从 localStorage 读取数据并显示
- 刷新后数据不丢失

请生成完整的 JavaScript 代码。
```

## 六、阶段五：优化打磨

> 这个阶段做细节调整，让 Demo 达到"发布级"质量。

### 6.1 需要检查和优化的内容

| 项目 | 检查内容 | 优化方向 |
|---|---|---|
| **响应式适配** | 在手机上打开是否正常 | 调整布局、字体大小 |
| **动效流畅度** | 动效是否卡顿 | 减少 animation 数量、简化 transition |
| **交互反馈** | 点击是否有明确反馈 | 增强点击动效、添加声音（可选） |
| **边界情况** | 时间超过24h怎么办 | 添加边界判断 |
| **视觉细节** | 间距、阴影、圆角是否一致 | 调整 CSS 变量 |
| **加载速度** | 页面是否加载快 | 压缩图片、减少外部依赖 |

### 6.2 TRAE 对话策略

**对话示例**：

```
请优化以下内容：

1. 响应式适配：在手机上打开时，卡片排列改为单列，字体适当缩小
2. 边界情况：如果某一天的时间总和超过24h，显示警告
3. 视觉细节：所有卡片圆角统一为12px，阴影统一为 box-shadow: 0 2px 8px rgba(0,0,0,0.08)

请生成优化后的代码。
```

## 十一、Vibe Coding 的核心原则

| 原则 | 说明 |
|---|---|
| **分步对话** | 不要一次性生成整个 Demo，分界面、分功能对话 |
| **保留证据** | 每个 Session ID、每张截图都要保留 |
| **验证每一步** | 每次对话后用浏览器打开验证效果 |
| **迭代优化** | 不满意就继续对话调整，不要直接改代码 |
| **保持风格一致** | 所有对话都要强调设计风格，避免偏离 |
