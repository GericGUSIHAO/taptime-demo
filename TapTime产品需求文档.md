# TapTime 产品需求文档（PRD）

> **文档版本**：v1.0
> **创建时间**：2026-07-03
> **产品名称**：TapTime
> **产品形态**：移动端 App（Demo 用交互式网页展示）
> **一句话定位**：轻点一下，记录时间。知道自己怎么过的，成为想成为的人。

---

## 一、产品概述

### 1.1 产品定位

TapTime 是一个**基于真实时间分配的习惯养成工具**。

不是又一个打卡 App，不是又一个时间记录 App。
它解决的核心问题是：**大多数习惯工具假设用户每天都有固定时间和意志力，但现实生活根本不是这样。**

### 1.2 核心差异化

| 传统工具 | TapTime |
|---|---|
| 先定习惯，再找时间 | 先看时间，再安排习惯 |
| 每天都要打卡，断了就焦虑 | 习惯养成后反向打卡，断了不焦虑 |
| 记录很详细，维护成本高 | 点一下就完成，15 秒内搞定 |
| 功能堆砌，大而全 | 三个核心功能，克制到极致 |

### 1.3 核心理念

> **不是靠意志力坚持，是靠时间和系统。**
> 打卡不是目的，养成习惯才是目的。
> 让工具服务人，而不是让人伺候工具。

## 二、目标用户

### 2.1 用户画像

| 维度 | 描述 |
|---|---|
| **人群** | 25-35 岁职场人，想改变但没方法 |
| **特征** | 学了很多方法论但生活没变化；下载过很多效率 App 但都用不久 |
| **痛点** | 打卡会断、时间记录太繁琐、Flag 年年立年年倒 |
| **心理** | 不是不自律，是那些工具本身就设计错了 |

### 2.2 用户场景

| 时间 | 行为 | 耗时 |
|---|---|---|
| **早上** | 打开 App，看一眼今天有哪些事 | 约 30 秒 |
| **白天** | 做完一件事，顺手点一下记录 | 约 3 秒 |
| **晚上** | 点一下记录今天做了什么，看看数据 | 约 1 分钟 |

**每天 1-2 次，每次不超过 2 分钟。**

---

## 三、核心痛点 & 解决方案

### 3.1 痛点

| 痛点 | 描述 | 本质问题 |
|---|---|---|
| **打卡永远会断** | 坚持 30 天，第 31 天忘了，就再也不想打开 | 打卡靠"每天记得"，不是"习惯真养成了" |
| **时间记录太繁琐** | 每件事都要记开始时间、结束时间、事情名称 | 记录本身变成了负担 |
| **计划不可行** | "每天健身 1 小时"——但根本没有那么多时间 | 习惯没绑定真实时间 |

### 3.2 解决方案

| 痛点 | TapTime 的解决方案 |
|---|---|
| 打卡会断 | **反向打卡**：习惯养成后（30 天），默认你做了，只有没做才需要点 |
| 记录太繁琐 | **极简记录**：点一下卡片 = 15 分钟，点两下 = 30 分钟，不需要写任何字 |
| 计划不可行 | **先看时间**：先记录一周时间，看看时间花在哪，再安排习惯 |

## 四、Demo 功能需求（MVP）

### 4.1 功能范围

> Demo 聚焦核心功能，让评审能体验完整的"记录时间 → 安排习惯 → 反向打卡"流程。

| 功能 | 说明 | 优先级 |
|---|---|---|
| **时间块记录** | 点一下卡片 = 15 分钟，点两下 = 30 分钟 | ✅ P0 核心 |
| **今日视图** | 显示今天记录的时间块和习惯 | ✅ P0 核心 |
| **习惯打卡** | 建立期正向打卡 + 稳定期反向打卡 | ✅ P0 核心 |
| **周视图** | 本周时间分布的简单可视化 | ✅ P1 重要 |
| **习惯生命周期展示** | 建立期 → 稳定期 → 记录期的状态切换 | 🔮 P2 可选 |
| **数据统计** | 时间分类占比、习惯完成率 | 🔮 P2 可选 |

### 4.2 不做的事（Scope 边界）

- ❌ 不做 AI 对话功能（Demo 阶段用预设数据模拟）
- ❌ 不做 SOP/例程管理（太复杂，初赛不需要）
- ❌ 不做社交/督促功能（增加压力）
- ❌ 不做番茄钟/日历集成（参考滴答清单的教训）
- ❌ 不做付费/会员功能（先让用户认可产品）

#### 5.2.2 交互需求

**时间块卡片交互**：

| 状态 | 显示 | 操作 | 说明 |

- 顶部显示"今天记录了 Xh Ymin"
- 每次点击时间块卡片，数字有递增动画

## 六、习惯生命周期设计

### 6.1 三阶段模型

| 阶段 | 时间 | 用户操作 | 系统行为 | 视觉表现 |
|---|---|---|---|---|
| **建立期** | 前 30 天 | 每天点一下"做了" | 显示"去打卡" | 空心圆 ○ → 实心圆 ● |
| **稳定期** | 30 天后 | 只有"没做"才需要点 | 默认"已完成" | 灰色 ✓，点击变 ✗ |
| **记录期** | 持续 | 几乎不需要操作 | 自动记录用于复盘 | 在周视图中统计 |

### 6.2 Demo 中的处理

> Demo 没有真实的 30 天时间，需要用预设数据展示。

- 预设习惯 1（冥想）：建立期第 12 天 → 展示正向打卡
- 预设习惯 2（阅读）：建立期第 5 天 → 展示正向打卡
- 预设习惯 3（运动）：稳定期第 45 天 → 展示反向打卡

**这样评审能在同一个 Demo 中体验到两种打卡模式的差异。**

## 七、交互设计规范

### 7.1 核心交互原则

| 原则 | 说明 |
|---|---|
| **零学习成本** | 打开就知道怎么用，不需要教程 |
| **3 秒内完成** | 核心操作（记录时间/打卡）最多 3 次点击 |
| **即时反馈** | 每次点击都有视觉反馈 |
| **仪式感** | 记录不是负担，是有温度的仪式 |

### 7.2 动效需求

| 动效 | 触发 | 时长 | 说明 |
|---|---|---|---|
| **数字递增动画** | 记录时间块时 | 0.3s | 顶部数字从旧值滚到新值 |
| **卡片点击反馈** | 点击时间块卡片 | 0.2s | 卡片缩放 + 高亮 |
| **打卡切换动画** | 点击习惯圆圈 | 0.2s | 圆圈填充/取消的过渡 |
| **页面切换** | Tab 切换 | 0.3s | 左右滑动 + 淡入淡出 |
| **滚动渐入** | 元素进入视口 | 0.4s | 从下方淡入 |
| **连续打卡庆祝** | 完成连续 7 天 | 0.5s | 微妙的粒子效果或光效 |

### 7.3 手势支持

| 手势 | 作用 |
|---|---|
| **点击** | 记录时间块 / 打卡 |
| **左滑** | 删除习惯条目 |
| **长按** | 进入编辑模式（习惯管理页） |

## 八、设计规范

### 8.1 视觉风格

**Editorial Magazine 极简风** — 参考欧美设计师品牌官网和艺术类 App 的设计语言。

**关键词**：极简、留白、有温度、仪式感

### 8.2 配色方案

| 色彩 | 色值 | 用途 |
|---|---|---|
| **米白底** | #FAF9F7 | 背景色 |
| **纯黑** | #1A1A1A | 主文字 |
| **深赭石** | #C45C2E | 强调色（按钮、选中态、数字） |
| **浅灰** | #E5E5E5 | 分割线、卡片边框 |
| **中灰** | #999999 | 次要文字 |
| **深灰** | #333333 | 卡片背景（暗色卡片） |

### 8.3 字体

| 字体 | 用途 | 说明 |
|---|---|---|
| **Fraunces** | 大标题、数字 | 衬线体，有杂志编辑感 |
| **Space Grotesk** | 正文、UI 文字 | 无衬线，干净现代 |

### 8.4 间距系统

| Token | 值 | 用途 |
|---|---|---|
| xs | 4px | 微间距（图标和文字间距） |
| sm | 8px | 紧凑间距 |
| md | 16px | 标准间距（卡片内 padding） |
| lg | 24px | 宽松间距（卡片间距） |
| xl | 32px | 大间距（区块间距） |
| 2xl | 48px | 超大间距（页面顶部/底部留白） |

### 8.5 圆角

| 元素 | 圆角 |
|---|---|
| 卡片 | 12px |
| 按钮 | 8px |
| 头像/图标 | 50%（圆形） |
| 全屏容器 | 0px |

### 8.6 组件规范

| 组件 | 规格 |
|---|---|
| **时间块卡片** | 96px × 96px，圆角 12px，点击时缩放 0.95 |
| **习惯条目** | 通栏，高 56px，左侧圆圈 24px |
| **底部导航** | 高 56px，3 个 Tab，图标 + 文字 |
| **数字展示** | Fraunces 字体，28px，赭石色 |

| 数据展示 | 周视图能让人"一眼看懂这周怎么过的" |
