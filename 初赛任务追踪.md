#### 1. Demo 简介
- **是什么**：一款极简的效率 App（Demo 为网页版）
- **面向谁**：想改变但没方法的职场人
- **主要功能**：
  - 轻点记录时间块（点1次=15min，短时间内多点自动合并为一条 n×15min 记录）
  - 本周时间块可视化（按类别着色，看到时间的形状）
  - 理想周排程（按天设置时间块，空白处显示可安排时间）

## 十、心态管理

### 10.1 不要做的事情

❌ **不要追求完美**
- 功能少没关系，能跑起来比什么都重要
- 设计不一定要惊艳，有自己的风格就好

❌ **不要同时做太多功能**
- 每次只做一件事
- 做完一件，测试通过，再做下一件

❌ **不要怕犯错误**
- 代码有 bug 很正常
- AI 会帮你修
- 每一个错误都是学习的机会

### 10.2 应该做的事情

✅ **每天都做点什么**
- 不一定要写很多代码
- 哪怕只添加一个按钮，也是进步
- 保持节奏，比偶尔爆发更重要

✅ **随时用 AI 帮忙**
- 不会写的代码？问 AI
- 不知道怎么设计？让 AI 给几个方案
- AI 是你的无限免费助手

✅ **让别人看你的东西**
- 不要藏着掖着
- 越早让别人用，越早发现问题

## 十二、遇到困难怎么办？

| 问题 | 解决方法 |
|---|---|
| 不知道怎么写代码 | 让 AI 帮你写。把需求说清楚，它会帮你。 |
| AI 写的代码有 bug | 把错误信息复制给 AI，问它怎么修 |
| 不知道怎么实现某个功能 | 让 AI 给你3个方案，选最简单的那个 |
| 想加的功能太多 | 回头看 MVP 功能清单，只做核心功能 |
| 进度太慢，焦虑 | 回到最基本的：每天1个小功能，能完成就行 |

## 十三、TRAE 对话 Prompt 模板

> 分界面、分功能对话，不要一次性生成整个 Demo

### 13.1 静态原型生成

**首页（Today）静态原型**

```
我要做一个效率类 App 的网页 Demo，产品名是 TapTime。

请帮我生成首页（Today）的静态原型，包含：

1. **今日时间块区域**：
   - 顶部显示日期和总时长
   - 显示6个预设卡片（工作、阅读、运动、休息、吃饭、游戏）
   - 每个卡片显示类别图标和已记录的时间
   - 点击交互：点1次=15min，点2次=30min，长按可精确设置

2. **今日记录列表**：
   - 显示今天记录的时间块列表
   - 每条记录显示类别、时长、时间

设计风格：
- Editorial Magazine 极简风
- 配色：米白底（#FAF9F7）、纯黑文字（#1A1A1A）、深赭石强调（#C45C2E）
- 字体：Fraunces（标题/数字）+ Space Grotesk（正文）
- 大量留白、圆角卡片、微阴影

先只做静态界面，不需要实际功能。请生成完整的 HTML/CSS 代码。
```

### 13.2 动效添加

```
请给刚才生成的首页添加动效：

1. 页面加载时，所有卡片依次渐入（从上到下，间隔100ms）
2. 点击时间卡片时，卡片缩放（scale 0.95 → 1），时间数字动画增长
3. 数字增长动画：从旧值滚动到新值，时长0.3s
4. 所有可点击元素悬停时轻微放大（scale 1.02）

请用 CSS animation 和 transition 实现，保持代码简洁。
```

### 13.3 功能实现

**时间块记录功能**

```
请实现时间块记录功能：

逻辑：
1. 点击卡片第1次 → 记录 15min
2. 点击卡片第2次 → 记录 30min（累加）
3. 点击卡片第3次 → 记录 45min（累加）
4. 长按卡片 → 弹出精确时间设置弹窗
5. 点击后数字动画增长到新值，总时长同步更新

数据存储：
- 用 localStorage 存储，键名 "taptime_records"
- 存储格式：{ "2026-07-10": [{ id, category, duration, time }] }

显示：
- 页面加载时从 localStorage 读取数据并显示
- 刷新后数据不丢失
- 记录列表按时间倒序排列

请生成完整的 JavaScript 代码。
```

## 十五、Vibe Coding 核心原则

| 原则 | 说明 |
|---|---|
| **分步对话** | 不要一次性生成整个 Demo，分界面、分功能对话 |
| **保留证据** | 每个 Session ID、每张截图都要保留 |
| **验证每一步** | 每次对话后用浏览器打开验证效果 |
| **迭代优化** | 不满意就继续对话调整，不要直接改代码 |
| **保持风格一致** | 所有对话都要强调设计风格，避免偏离 |
| **截图要清晰** | 截图要包含 TRAE 的界面，证明是用 TRAE 开发的 |
| **截图要标注** | 每张截图要说明展示的是什么 |
