## 2.2 UI 设计

| 任务 | 状态 | 说明 |
|---|---|---|
| 研究 10 个简洁好看的 APP 界面 | ☐ | 可以用 Mobbin，也可以自己在手机上截图 |
| 挑选 3 个喜欢的配色方案 | ☐ | 让 AI 帮你根据已有的 APP 生成 |

- 保持极简，功能越少越好做

**目标**：有 3-4 张界面设计图，可以开始开发了。

# 第三阶段：技术选型与开发环境（2天内完成）

## 3.1 选择技术栈

**推荐的最简单方案**（从易到难排序）：

| 方案 | 难度 | 说明 | 适合 |
|---|---|---|---|
| **纯网页 APP** | ⭐ 最简单 | HTML/CSS/JS，用浏览器就能用 | MVP快速验证 |
| **React + Vite** | ⭐⭐ 简单 | 现在最主流的前端框架 | 长期维护 |
| **Next.js** | ⭐⭐⭐ 中等 | 服务端渲染，功能更强大 | 需要后端功能 |

**我的建议**：从纯网页 APP 开始。

**为什么**：
- 最简单，AI 最容易帮你
- 不需要学习复杂框架
- 先做出能用的东西，再考虑升级
- 大赛只需要一个 demo，网页 APP 完全够用

**后续升级路径**：
- 网页版 → 添加移动端适配 → 添加 PWA（可以装到手机桌面） → 后期用 React Native 或 SwiftUI 做原生 APP

---

## 3.2 开发环境准备

| 任务 | 状态 | 说明 |

# 第四阶段：MVP 开发（核心阶段，7-14天）

> 核心原则：每天做一点，不要追求完美，先做出来再说。

## 开发顺序（从简单到复杂）

**目标**：每天做 1-2 个小功能，不要一天想做太多。

### Day 1-2：今日视图（最核心界面）

| 任务 | 状态 | 说明 |
|---|---|---|
| 做出今日视图的基本结构 | ☐ | 顶部概览，下面是卡片列表 |
| 让界面看起来好看 | ☐ | 配色/间距/字体，让 AI 帮你调 |
| 添加"添加待办"功能 | ☐ | 一行文字输入，回车添加 |
| 添加"完成待办"功能 | ☐ | 点一下就完成，消失或者移到下面 |
| 测试一遍：添加3个待办 → 完成2个 → 刷新页面 | ☐ | 看数据是不是还在 |

**第一天结束时你应该有**：一个可以添加和完成待办的简单网页。

---

### Day 3-4：时间记录（核心价值功能）

| 任务 | 状态 | 说明 |
|---|---|---|
| 做出"预设卡片"的界面 | ☐ | 健身/阅读/工作/休息 4个卡片 |
| 实现"点一下15分钟，两下30分钟"的逻辑 | ☐ | 这是你的独特功能，要做对 |
| 点击卡片后开始计时 | ☐ | 自动计算结束时间 |
| 显示当前计时状态 | ☐ | 比如："正在阅读，还剩28分钟" |
| 时间到了给个简单提示 | ☐ | 弹出提示或声音提示 |

**第二天结束时你应该有**：点击卡片就能计时，时间到了有提示。

---

### Day 5-6：习惯功能

| 任务 | 状态 | 说明 |
|---|---|---|
| 添加"今日习惯"区域 | ☐ | 列出今天有哪些习惯要完成 |
| 点一下完成习惯 | ☐ | 跟完成待办一样简单 |
| 可以添加新的习惯（在iPhone端设置） | ☐ | 简单的设置界面 |
| 习惯统计（今天完成了几个） | ☐ | 在概览区域显示 |

**第三天结束时你应该有**：一个可以记录习惯的功能。

---

### Day 7-8：今日概览 + 数据统计

| 任务 | 状态 | 说明 |
|---|---|---|
| 顶部概览显示今日数据 | ☐ | 完成待办数量 / 计时总时长 / 习惯完成数 |
| 添加"本周统计"页面 | ☐ | 简单的数字，不需要复杂图表 |
| 添加"历史记录"页面 | ☐ | 能看到过去几天做了什么 |

**第四天结束时你应该有**：基本的数据统计功能。

---

### Day 9-10：交互优化 + 移动端适配

| 任务 | 状态 | 说明 |
|---|---|---|
| 在手机上测试，调整大小和布局 | ☐ | 用手机浏览器打开网页测试 |
| 添加一些动画和过渡效果 | ☐ | 不要太复杂，点一下有反应就行 |
| 确保触摸区域足够大（手指能点到） | ☐ | 最小44x44像素 |
| 添加 PWA 功能（可选） | ☐ | 可以"添加到主屏幕"，像APP一样 |

**第五天结束时你应该有**：一个在手机上也能用的网页APP。

---

### Day 11-14：Apple Watch 版本（可选，时间不够可以跳过）

| 任务 | 状态 | 说明 |
|---|---|---|
| 学习 watchOS 开发基础 | ☐ | 看1-2个教程视频 |
| 做出 Watch 界面：预设卡片 + 今日习惯 | ☐ | 用 SwiftUI，让 AI 帮你写 |
| 实现与手机的数据同步 | ☐ | WatchConnectivity |
| 测试在 Watch 上点击计时 | ☐ | 确认交互流畅 |

**重要**：如果时间不够，这个阶段可以跳过。
**大赛初赛只需要一个可运行的 demo 就行，Watch 版本是加分项不是必须项。**

# 最重要的心态管理

## 不要做的事情

❌ **不要学完所有教程再动手**
- 看1个案例视频就开始
- 边做边学，不是先学再做

❌ **不要追求完美**
- 第一个版本丑没关系
- 功能少没关系
- 能跑起来比什么都重要

❌ **不要同时做太多功能**
- 每次只做一件事
- 做完一件，测试通过，再做下一件

❌ **不要怕犯错误**
- 代码有 bug 很正常
- AI 会帮你修
- 每一个错误都是学习的机会

❌ **不要和别人比**
- 你是在做自己的产品
- 别人做得快不代表什么
- 你只要把自己的产品做好

---

## 应该做的事情

✅ **每天都做点什么**
- 不一定要写很多代码
- 哪怕只添加一个按钮，也是进步
- 保持节奏，比偶尔爆发更重要

✅ **随时用 AI 帮忙**
- 不会写的代码？问 AI
- 不知道怎么设计？让 AI 给几个方案
- 不会配色？让 AI 选
- AI 是你的无限免费助手

✅ **让别人看你的东西**
- 不要藏着掖着
- 越早让别人用，越早发现问题
- 别人的反馈是最好的成长

✅ **享受过程**
- 你在做一个自己用的产品
- 这个过程本身就很有价值
- 能做成什么样，就做到什么样
- 做出来，就是胜利

# 每日流程建议

## 每天开始时（10分钟）

1. 打开这个行动清单，看今天应该做什么
2. 打开项目，看看昨天做到哪了
3. 确定今天要完成的1件事（最多2件）

## 每天结束时（10分钟）

1. 测试今天写的功能，确保能用
2. 更新这个行动清单的完成状态
3. 写一句话："今天完成了xxx"

**不需要写很多，能坚持下来才重要。**

# 遇到困难怎么办？

## 常见问题和解决方法

| 问题 | 解决方法 |
|---|---|
| 不知道怎么写代码 | 让 AI 帮你写。把需求说清楚，它会帮你。 |
| AI 写的代码有 bug | 把错误信息复制给 AI，问它怎么修 |
| 不知道怎么实现某个功能 | 让 AI 给你3个方案，选最简单的那个 |
| 想加的功能太多 | 回头看 MVP 功能清单，只做核心功能 |
| 进度太慢，焦虑 | 回到最基本的：每天1个小功能，能完成就行 |
| 不知道好不好看 | 让 AI 帮你调配色和布局，多试几个方案 |
| 不知道用户体验好不好 | 让朋友用一下，不问他们好不好用，只观察他们会不会用 |
