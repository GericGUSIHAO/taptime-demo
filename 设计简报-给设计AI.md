# TapTime 设计简报（Design Brief）

> 本文档用于交给设计类AI，帮助其完全理解产品理念并产出高质量的UI设计/网页设计。

---

## 一、项目概述

### 1.1 产品名称
**TapTime（时刻）**

### 1.2 一句话定位
**轻点一下，记录一周时间。知道自己怎么过的，成为想成为的人。**

### 1.3 产品本质
一个**基于真实时间分配的习惯养成工具**。

不是又一个打卡App，不是又一个时间记录App。
它解决的核心问题是：**大多数习惯工具假设用户每天都有固定时间和意志力，但现实生活根本不是这样。**

### 1.4 核心差异化

| 传统工具 | TapTime |
|---|---|
| 先定习惯，再找时间 | 先看时间，再安排习惯 |
| 每天都要打卡，断了就焦虑 | 习惯养成后反向打卡，断了不焦虑 |
| 记录很详细，维护成本高 | 点一下就完成，15秒内搞定 |
| 功能堆砌，大而全 | 三个核心功能，克制到极致 |

### 1.5 核心理念

> **不是靠意志力坚持，是靠时间和系统。**
> 打卡不是目的，养成习惯才是目的。
> 让工具服务人，而不是让人伺候工具。

## 二、目标用户

### 2.1 用户画像

| 维度 | 描述 |
|---|---|
| **人群** | 25-35岁职场人，想改变但没方法 |
| **特征** | 学了很多方法论但生活没变化；下载过很多效率App但都用不久 |
| **痛点** | 打卡会断、时间记录太繁琐、Flag年年立年年倒 |
| **心理** | 不是不自律，是那些工具本身就设计错了 |

### 2.2 使用场景

| 时间 | 行为 | 耗时 |
|---|---|---|
| **早上** | 打开App，看一眼今天有哪些事 | 约30秒 |
| **白天** | 做完一件事，顺手点一下记录 | 约3秒 |
| **晚上** | 点一下记录今天做了什么，看看数据 | 约1分钟 |

**每天1-2次，每次不超过2分钟。**

---

## 三、核心痛点与解决方案

### 3.1 三大痛点

| 痛点 | 描述 | 本质问题 |
|---|---|---|
| **打卡永远会断** | 坚持30天，第31天忘了，就再也不想打开 | 打卡靠"每天记得"，不是"习惯真养成了" |
| **时间记录太繁琐** | 每件事都要记开始时间、结束时间、事情名称 | 记录本身变成了负担 |
| **计划不可行** | "每天健身1小时"——但根本没有那么多时间 | 习惯没绑定真实时间 |

### 3.2 TapTime的解决方案

| 痛点 | 解决方案 |
|---|---|
| 打卡会断 | **反向打卡**：习惯养成后（30天），默认你做了，只有没做才需要点 |
| 记录太繁琐 | **极简记录**：点一下卡片=15分钟，点两下=30分钟，不需要写任何字 |
| 计划不可行 | **先看时间**：先记录一周时间，看看时间花在哪，再安排习惯 |

## 四、设计风格

### 4.1 整体风格

**Editorial Magazine 极简风** — 参考欧美设计师品牌官网和艺术类App的设计语言。

**关键词**：极简、留白、有温度、仪式感、克制、高级

**感觉**：
- 像一本精致的杂志，不是一个工具App
- 像手写日记一样有温度，不是冷冰冰的数据面板
- 像设计师品牌官网一样克制，不是功能堆砌的产品

### 4.2 配色方案

| 色彩 | 色值 | 用途 |
|---|---|---|
| **米白底** | #FAF9F7 | 背景色（主背景） |
| **纯黑** | #1A1A1A | 主文字、标题 |
| **深赭石** | #C45C2E | 强调色（按钮、选中态、数字、重点） |
| **浅灰** | #E5E5E5 | 分割线、卡片边框、未选中态 |
| **中灰** | #999999 | 次要文字、辅助信息 |
| **深灰** | #333333 | 卡片背景（暗色卡片） |

**补充色（用于分类）**：
- 工作：#C45C2E（赭石色）
- 阅读：#4A6FA5（雾霾蓝）
- 运动：#5B9A6F（苔藓绿）
- 休息：#8B7355（卡其棕）
- 吃饭：#D4A373（奶茶色）
- 游戏：#8B5CF6（紫罗兰）

### 4.3 字体

| 字体 | 用途 | 说明 |
|---|---|---|
| **Fraunces** | 大标题、数字、日期 | 衬线体，有杂志编辑感，优雅有质感 |
| **Space Grotesk** | 正文、UI文字、按钮 | 无衬线，干净现代，易读性好 |

**字体层级**：
- 大标题：Fraunces, 24-28px, 600-700 weight
- 次级标题：Space Grotesk, 16px, 500 weight
- 正文：Space Grotesk, 14px, 400 weight
- 辅助文字：Space Grotesk, 12-13px, 400 weight
- 数字展示：Fraunces, 20-28px, 600-700 weight

### 4.4 间距系统

| Token | 值 | 用途 |
|---|---|---|
| xs | 4px | 微间距（图标和文字间距） |
| sm | 8px | 紧凑间距 |
| md | 16px | 标准间距（卡片内padding） |
| lg | 24px | 宽松间距（卡片间距） |
| xl | 32px | 大间距（区块间距） |
| 2xl | 48px | 超大间距（页面顶部/底部留白） |

### 4.5 圆角规范

| 元素 | 圆角 |
|---|---|
| 卡片（主要） | 12-16px |
| 按钮 | 8-12px |
| 头像/图标 | 50%（圆形） |
| 全屏容器 | 0px |
| 底部弹出层 | 24px（顶部圆角） |

### 4.6 阴影规范

| 层级 | 阴影 | 用途 |
|---|---|---|
| 轻量 | 0 1px 3px rgba(0,0,0,0.04) | 卡片、按钮默认态 |
| 中等 | 0 4px 12px rgba(0,0,0,0.08) | 悬浮态、弹出层 |
| 深度 | 0 10px 40px rgba(0,0,0,0.12) | 模态框、重要浮层 |

## 五、页面结构与功能

### 5.1 整体架构

```
TapTime
├── Today（今日）← 主页面（底部Tab第一个）
│   ├── 顶部：日期 + 今日概览
│   ├── 时间记录列表：今天记录的时间块
│   └── 底部按钮：轻点记录
│
├── Week（本周）← 第二个页面（右滑/底部Tab切换）
│   ├── 本周时间分布条形图
│   ├── 本周打卡日历
│   └── 本周总计
│
└── 记录弹窗（底部弹出）
    ├── 选择类别（6个图标卡片）
    └── 选择时长（15m/30m/45m/1h/1.5h/2h）
```

### 5.2 页面一：Today（今日）

**这是用户每天打开的主页面。**

**页面结构（从上到下）**：
1. **顶部日期区**
   - 日期："7月8日 周二"（Fraunces，24px，加粗）
   - 副标题："今天"（14px，灰色）
   - 留白要大，有杂志标题的感觉

2. **今日记录列表**
   - 按类别分组显示今天的记录
   - 每条记录：图标 + 类别名称 + 次数 + 总时长
   - 卡片式布局，白色背景，轻微阴影
   - 时长数字用Fraunces字体，对应类别的颜色
   - 如果没有记录，显示引导文案："今天还没有记录，点击下方按钮开始"

3. **底部合计区**
   - 分割线
   - 左侧："共记录"（灰色小字）
   - 右侧：总时长（Fraunces大字体，赭石色）

4. **主按钮**
   - 全宽黑色按钮："轻点记录"
   - 白色文字，圆角16px
   - 点击有缩放反馈

### 5.3 页面二：Week（本周）

**展示本周的时间分布，让用户"知道自己怎么过的"。**

**页面结构（从上到下）**：
1. **顶部标题区**
   - 标题："本周"（Fraunces，24px，加粗）
   - 副标题：日期范围（"7月7日 - 7月13日"，14px，灰色）

2. **时间分布图**
   - 白色卡片背景，圆角16px
   - 每个类别一行：图标+名称 | 时长数字 | 横向进度条
   - 进度条颜色对应类别颜色
   - 进度条有动画效果（从0到目标值的过渡）

3. **本周记录日历**
   - 标题："本周记录"（14px，灰色）
   - 7个圆点代表周一到周日
   - 有记录：实心赭石色圆
   - 部分记录：空心灰色圆
   - 没记录：浅灰背景或不显示
   - 下方显示连续天数

4. **本周总计**
   - 白色卡片背景
   - 左侧："共记录"（灰色小字）
   - 右侧：总时长（Fraunces大字体，黑色）

### 5.4 记录弹窗（底部弹出）

**点击"轻点记录"按钮后从底部滑出。**

**结构**：
1. **标题**："在做什么？"（Fraunces，20px，居中）

2. **类别选择**
   - 3×2网格布局，6个类别卡片
   - 每个卡片：图标 + 名称
   - 默认状态：浅灰边框，米白背景
   - 选中状态：2px彩色边框（对应类别颜色），白色背景
   - 点击有缩放反馈

3. **时长选择**
   - 标题："多久？"（居中，灰色小字）
   - 6个按钮水平排列：15m、30m、45m、1h、1.5h、2h
   - 默认状态：白色背景，黑色文字，灰色边框
   - 选中状态：黑色背景，白色文字
   - 按钮宽度均分，高度一致

4. **确认按钮**
   - 全宽黑色按钮："记上"
   - 未选中时：灰色不可点击状态
   - 选中类别和时长后：黑色可点击状态

## 六、交互设计规范

### 6.1 核心交互原则

| 原则 | 说明 |
|---|---|
| **零学习成本** | 打开就知道怎么用，不需要教程 |
| **3秒内完成** | 核心操作（记录时间/打卡）最多3次点击 |
| **即时反馈** | 每次点击都有视觉反馈 |
| **仪式感** | 记录不是负担，是有温度的仪式 |

### 6.2 动效需求

| 动效 | 触发 | 时长 | 缓动曲线 | 说明 |
|---|---|---|---|---|
| **数字递增动画** | 记录时间块时 | 0.3s | ease-out | 顶部数字从旧值滚到新值 |
| **卡片点击反馈** | 点击时间块卡片 | 0.2s | ease-in-out | 卡片缩放（0.95）+ 高亮 |
| **打卡切换动画** | 点击习惯圆圈 | 0.2s | ease-in-out | 圆圈填充/取消的过渡 |
| **页面切换** | Tab切换 | 0.3s | ease-out | 左右滑动 + 淡入淡出 |
| **滚动渐入** | 元素进入视口 | 0.4s | ease-out | 从下方淡入（translateY: 10px → 0） |
| **底部弹窗出现** | 点击记录按钮 | 0.3s | ease-out | 从底部滑出 + 背景遮罩淡入 |
| **进度条动画** | 进入周视图 | 0.5s | ease-out | 进度条从0增长到目标值 |

### 6.3 手势支持

| 手势 | 作用 |
|---|---|
| **点击** | 记录时间块 / 打卡 / 切换Tab |
| **左右滑动** | 在今日和本周之间切换 |
| **下拉** | 刷新/重新加载（可选） |

### 6.4 按钮状态

| 状态 | 视觉表现 |
|---|---|
| **Default（默认）** | 正常样式 |
| **Hover（悬浮）** | 轻微缩放（0.98）+ 阴影加深 |
| **Active（按下）** | 缩放至0.95 + 颜色加深 |
| **Disabled（禁用）** | 灰色背景 + 灰色文字 + 不可点击光标 |

## 七、设计参考与灵感来源

### 7.1 竞品参考（学习优点，避免缺点）

| 产品 | 学习的点 | 避免的点 |
|---|---|---|
| **Nosh** | 日记本+贴纸视觉、极简交互、仪式感、好看让人愿意用 | - |
| **HabitDone** | 简洁克制、反向打卡逻辑、不花哨 | 功能太简单 |
| **MarkTimes** | 时间块记录方式、设计美感 | 易用性差，学习成本高 |
| **滴答清单** | - | 功能太多太杂、一上来就推会员、信息密度过高 |

### 7.2 设计风格参考

- **Aesop官网**：极简、留白、高级感
- **Kinfolk杂志**：编辑感、排版、温度
- **苹果iOS设计语言**：毛玻璃、圆角、流畅动效
- **无印良品**：克制、实用、不花哨

### 7.3 设计灵感关键词

- Editorial design（编辑设计）
- Quiet luxury（静奢风）
- Slow design（慢设计）
- Warm minimalism（温暖极简）
- Journal/diary aesthetic（日记本美学）

## 十、品牌调性与文案风格

### 10.1 品牌调性

- **温和但坚定**：不说教，不鸡血，像朋友在建议
- **克制不炫耀**：好的设计是看不见的设计
- **有温度**：不是冷冰冰的工具，是陪伴成长的伙伴
- **真实不虚假**：不承诺"30天改变人生"，只说"慢慢来，比较快"

### 10.2 文案风格

- 简短，不啰嗦
- 口语化，不官方
- 有画面感，不抽象
- 像和朋友聊天，不是产品说明书

**好的例子**：
- "轻点一下，记录一周时间"
- "知道自己怎么过的，成为想成为的人"
- "今天还没有记录，点击下方按钮开始"

**不好的例子**：
- "提升效率，实现自我管理"
- "您尚未创建任何记录"
- "点击此处开始您的高效之旅"
